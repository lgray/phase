## Root cause of the swallows
`swallow_check::check_swallowed_clauses` computes `cleaned = strip_parens(lower)` — does NOT strip the ability-word prefix (`swallow_check.rs:100`). `detect_condition_if` (`swallow_check.rs:2093`) fires on marker `stripped.contains(" if ")` (`:2244`) unless the AST JSON carries a `cond_markers` entry (`:2258`, incl. `"\"condition\":{"`). A bare leading `"If a Wizard…"` at position 0 → no `" if "` → never trips. But `"Flare Star — if a wizard…"` / `"Delirium — if a source…"` inject the space via `— if` → false-positive `Condition_If`. The leading `"if [source] would deal … instead"` is CR 614.1a replacement syntax (docs:3056), represented by the ReplacementDefinition — NOT a game-state gate. Runtime honors all axes at `replacement.rs:4538`(condition)/`:4551`(source_filter)/`:4564`(combat_scope)/`:4578`(target_filter).
