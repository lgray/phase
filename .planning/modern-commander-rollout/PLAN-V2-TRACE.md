## 0. What the repair pass changed (read first)

The prior synthesis under-counted: its 86 named tranches summed to **Σstd = 179 / Σmc = 1168**, below the measured distinct-card floors (200 / 1366) and below the measured lane-membership sums (**216 / 1471**). A completeness critic proved the invariant "every card ships / 0 unclustered" was broken. Two mechanical defects, both now fixed:

1. **Un-carved residue inside covered lanes.** Five of the twelve report lanes had memberships no named tranche claimed — and in every case the residue was *already described in the corrections appendix but never entered the table*. The repair pass books each as an explicit residual tail: `Ttail-engine-aslongas` (**9/70** — the R2-aslongas-conditional-static parser class, appendix figure reproduced to the card), `Ttail-mod-reassign` (10/67), `Ttail-cond-reassign` (2/19), `Ttail-quantity-keywordN` (1/18 — the 17 keyword-N cards + Neriv, appendix-exact), `Ttail-effect-repeat-list` (1/22).
2. **Dropped marker lanes.** Twenty-one marker lanes (`file_attr` values) had **zero** named-tranche attribution. Seven were exclusively dropped (critic L14/L15/L17/L19/L29/L31/L33 = 3 std / 24 mc cards with no other home); four were borderline reassignment-only (L22/L24/L25/L27); ten were co-located with an existing fix-home but never booked. All 21 now carry an explicit tail tranche, ranked where its score lands.

**Accounting authority.** Coverage now keys on the TSV's **marker lane** = column 8 `file_attr` (the file `trace_cluster.py`'s `attribute_handler` grep-pinned each gap-row to). This is the *measured* attribution and is distinct from a tranche's *corrected fix-home* (where the fix actually lands). Named tranches keep their corrected fix-home for the work description; the reconciliation (§3) closes by marker lane. Per lane: **tail = max(0, measured membership − Σ named-tranche rows originating in that lane)**, computed mechanically over the TSV + the plan table.

## 1. Method


**Provenance.** Tranches synthesized from 12 per-lane attribution reports produced by `trace_cluster.py` @ `b7194fc30` over the parser-gaps analyzer output, coverage-data snapshot `2026-07-03T23:00Z`. Each lane report was an independent, measured attribution pass (sampled cards + `grep` of the real fix site) over one nominal hot file.

**Measured-lane model.** A tranche = one *class fix* landing in one *fix-home file*. Lanes serialize internally (tranches sharing a file must run one-at-a-time); lanes parallelize across disjoint files. A fix shared across both pools pays double (cross-pool overlap).

**Attribution corrections are load-bearing.** `trace_cluster.py`'s `attribute_handler` greps a bare head-token literal and takes the first walk-order hit, so several nominal "lanes" are grep-collision buckets, not real serialization units. The **fix-home file shown below is the corrected/measured home**, not the marker lane. Four lanes are dissolved (`oracle_dispatch.rs`, `search.rs`, `mana.rs`, `duration.rs`) and their tranches redistributed to their true hot files; two are corrected in place (`oracle_effect/mod.rs`, engine-runtime). See the Corrections appendix.

**Scoring (transparent).** `score = (std×2 + mc) × difficulty_mult`, `difficulty_mult = {cheap 1.5, moderate 1.0, heavy-infra 0.5}`. Standard weighted **2×** (smaller, release-facing pool). Heavy-infra halved because its member count is dominated by runtime, not the parser near-miss that unblocks it.

**Counts** are summed per-lane member counts as reported (gap-rows/cards), **not** de-duplicated unique cards across lanes: a card with gaps in two lanes is counted in each. Global totals: **86 tranches, Σstd = 179, Σmc = 1168.**

**Naming.** Two lanes independently numbered tranches `Tcond-*`. The `oracle_nom/condition.rs` lane keeps `Tcond-1..11`; the `oracle_effect/conditions.rs` lane (originally also `Tcond-1..6`) is relabeled here by suffix (`Tcond-otherwise-else-binding`, etc.) to disambiguate. The fix-home column is the authoritative key.

**In-flight OLD-plan work finishes under its existing definition.** The S25 40-card standard tranche and PR-6.75 are already scheduled under the OLD handler-regex plans; they complete as-defined and are **not** re-diced by this ranking. This ranking governs work started after they land.


**Repair-pass provenance.** The 26 tail tranches were derived by (a) mapping each named tranche to its originating marker lane by name prefix (`Timp-*`→`imperative.rs`, `Tcond-<n>`→`oracle_nom/condition.rs`, `Tcond-<word>`→`oracle_effect/conditions.rs`, `Tq-*`→`oracle_quantity.rs`, `Tsearch-*`→`search.rs`, `Ttrigger-*`→`oracle_trigger.rs`, `Tmod-*`→`mod.rs`, `Tmana-*`→`mana.rs`, `Tdur-*`→`duration.rs`, `T-dispatch-*`→`oracle_dispatch.rs`, `Trt-*`→`engine-runtime`, `Toe-*`→`oracle_effect/`), (b) subtracting Σ claimed from measured membership per lane, (c) booking the residual (report lanes) or full membership (non-report lanes) as a tail. Tail difficulty defaults to heavy/bespoke (×0.5) except where the appendix identifies a coherent parser class (`Ttail-engine-aslongas`, `-cond-reassign`, `-quantity-keywordN` → moderate ×1.0).

## 3. Reconciliation — Modern+Commander pool (closes exactly)

**Units.** A *membership* is a distinct `(card, marker-lane)` pair — a card with gaps in two marker lanes contributes two memberships (the plan's stated "not de-duplicated across lanes" rule, made exact). The measured authority, taken directly from the Modern+Commander TSV:

| Quantity | Value | Source |
|---|---:|---|
| Distinct cards (col 1 unique) | **1366** | measured — matches the stated floor |
| Lane memberships (distinct card × `file_attr`) | **1471** | measured — Σ of the per-lane column below |
| Multi-lane overlap (memberships − distinct) | **105** | 1471 − 1366 = 105 cards carry gaps in ≥2 marker lanes |

**Closing arithmetic (membership units):**

```
Σ measured membership          = 1471      (authority)
  = Σ effective-named (capped) = 1157
  + Σ tail tranches            = 314
  --------------------------------------
    1157 + 314                = 1471   ✓ EXACT
```

The named tranches carry the synthesis's **reported gap-row counts** (Σ = 1168); of these, **11** exceed their lane's distinct-card membership — the only figure the repair pass could not fold into a positive tail. They are *same-card multi-gap rows* (`oracle_effect/imperative.rs` +8 and `oracle_effect/search.rs` +3). Those lanes are **saturated** (tail 0); the 11-membership delta is disclosed here, not forced. Everything else reconciles to the card.

**"0 unclustered" verified:** every one of the 33 Modern+Commander marker lanes satisfies `membership = effective-named + tail`, so every `(card, lane)` membership is booked to exactly one tranche (named or tail). No lane is unrepresented; no card's gap is homeless.
